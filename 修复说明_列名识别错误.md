# 列名识别错误修复说明

**修复时间**: 2026-02-28  
**问题类型**: 意图库匹配错误，列名识别混乱  
**修复状态**: ✅ 已修复

---

## 一、问题描述

### 用户输入
```
按照各个省公司的险情确认时长平均值绘制柱状图
```

### 用户期望
- **X 轴（分组列）**: 省公司名称
- **Y 轴（数值列）**: 险情确认时长
- **聚合函数**: 平均值 (avg)
- **标题**: 各省公司险情确认时长平均值

### 系统实际生成（错误）
```json
{
  "chartType": "bar",
  "aggregateFunction": "avg",
  "sortOrder": "desc",
  "xAxisColumn": "险情确认时长",
  "yAxisColumn": "事发时间",
  "title": "各险情确认时长事发时间平均值",
  "description": "按照险情确认时长统计事发时间的平均值"
}
```

**问题**:
1. ❌ X 轴错误：应该是"省公司名称"，实际是"险情确认时长"
2. ❌ Y 轴错误：应该是"险情确认时长"，实际是"事发时间"
3. ❌ 标题混乱："各险情确认时长事发时间平均值"完全不符合语义

---

## 二、问题根因分析

### 2.1 意图库匹配问题

匹配到的意图模板：`avg_bar_implicit`（各分组平均值柱状图）

**原代码** (`intentLibrary.js` 第 115-116 行):
```javascript
extractParams: (text, columns) => {
    // 提取分组列和数值列
    const groupCol = this.findColumn(text, columns, ['省', '市', '区', '部门', '类别', '类型', '分组', '公司'], true);
    const valueCol = this.findColumn(text, columns, ['金额', '数值', '数量', '时长', '分数', '销量', '额', '时间'], false);
    // ...
}
```

**问题**:
1. `findColumn` 函数虽然会优先匹配完整列名，但没有**根据列名特征判断是分组列还是数值列**
2. "险情确认时长"包含"时"，被错误识别为分组列（因为关键词中有"时间"）
3. "事发时间"包含"时间"，被错误识别为数值列

### 2.2 正则模式缺失

用户输入"按照各个省公司的险情确认时长平均值"没有匹配到正确的正则模式。

**现有模式** (`queryConfigGenerator.js`):
```javascript
// 模式1: 按照 Y 统计 X 的平均值
regex: /按 [照]?(.+?) 统计 (.+?) 的 (平均值|平均|均值)/
// 匹配: "按照省公司统计险情确认时长的平均值" ✅
// 不匹配: "按照省公司的险情确认时长平均值" ❌
```

**缺失的模式**:
```javascript
// 应该支持: "按照省公司的险情确认时长平均值"
// 即: 按照 Y 的 X 平均值
```

## 三、修复方案

### 3.1 修复 intentLibrary.js

**改进 `extractParams` 函数** (`intentLibrary.js` 第 113-130 行):

```javascript
extractParams: (text, columns) => {
    // 改进：首先尝试从文本中直接提取提到的列名
    const lowerText = text.toLowerCase();
    
    // 按列名长度降序排序，优先匹配更长的列名
    const sortedColumns = [...columns].sort((a, b) => b.length - a.length);
    
    // 策略1：直接匹配完整列名（在文本中查找所有列名）
    let foundGroupCol = null;
    let foundValueCol = null;
    
    for (const col of sortedColumns) {
        const lowerCol = col.toLowerCase();
        if (lowerText.includes(lowerCol)) {
            // 判断这是分组列还是数值列
            const isGroupKeyword = ['省', '市', '区', '部门', '类别', '类型', '分组', '公司', '区域', '地区'].some(k => col.includes(k));
            const isValueKeyword = ['金额', '数值', '数量', '时长', '分数', '销量', '额', '时间', '长度', '大小', '多少'].some(k => col.includes(k));
            
            if (isGroupKeyword && !foundGroupCol) {
                foundGroupCol = col;
                console.log(`[IntentLibrary] 识别到分组列: ${col}`);
            } else if (isValueKeyword && !foundValueCol) {
                foundValueCol = col;
                console.log(`[IntentLibrary] 识别到数值列: ${col}`);
            }
        }
    }
    
    // 策略2：使用关键词匹配作为兜底
    const groupCol = foundGroupCol || this.findColumn(text, columns, ['省', '市', '区', '部门', '类别', '类型', '分组', '公司'], true);
    const valueCol = foundValueCol || this.findColumn(text, columns, ['金额', '数值', '数量', '时长', '分数', '销量', '额', '时间'], false);
    
    // ...
}
```

**改进点**:
1. ✅ 优先在文本中查找所有列名
2. ✅ 根据列名特征（是否包含分组关键词或数值关键词）判断列类型
3. ✅ 使用关键词匹配作为兜底策略

### 3.2 修复 queryConfigGenerator.js

**新增正则模式** (`queryConfigGenerator.js`):

```javascript
{
    // 按照 Y 的 X 平均值/总和（新增：支持"按照省公司的险情确认时长平均值"）
    regex: /按 [照]?(.+?) 的 (.+?)(平均值|平均|均值|总和|合计|总计)/,
    extract: (match, columns) => ({
        groupCol: this.findColumn(match[1], columns),
        valueCol: this.findColumn(match[2], columns),
        aggregateFunction: match[3].includes('平均') ? 'avg' : 'sum'
    })
},
```

**新增语法支持**:
- ✅ "按照省公司的险情确认时长平均值"
- ✅ "按照部门的预算总和"
- ✅ "按照区域的销售额平均值"

## 六、技术说明

### 列名识别策略

修复后的列名识别采用三级策略：

```
策略1: 直接匹配完整列名
        ↓ 在文本中查找所有列名
        ↓ 根据列名特征判断类型
        ↓ 优先识别分组列和数值列

策略2: 关键词匹配（兜底）
        ↓ 使用分组关键词匹配分组列
        ↓ 使用数值关键词匹配数值列

策略3: 默认处理
        ↓ 如果仍无法识别，使用第一列作为分组列
```

### 列名特征判断

**分组列特征**（包含以下关键词）:
- 省、市、区、部门、类别、类型、分组、公司、区域、地区

**数值列特征**（包含以下关键词）:
- 金额、数值、数量、时长、分数、销量、额、时间、长度、大小、多少

### 优先级说明

1. **优先匹配完整列名**: 避免部分匹配（如"时"匹配到"时间"）
2. **按列名长度排序**: 优先匹配更长的列名（如"省公司名称"优先于"省"）
3. **特征判断**: 根据列名内容判断是分组列还是数值列
