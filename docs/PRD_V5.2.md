## 1. 产品概述

### 1.1 产品定位
智能数据分析助手是一款面向非技术用户的自然语言数据分析工具。用户无需掌握SQL或编程知识，只需用自然语言描述分析需求，系统即可自动理解意图、执行查询、生成可视化图表，并提供智能分析建议。

### 1.2 核心价值主张
- **零门槛数据分析**：自然语言交互，无需学习成本
- **智能意图理解**：三层混合架构，准确识别15种意图类型
- **大数据支持**：自动切换数据库模式，支持百万级数据
- **一键可视化**：智能推荐最佳图表类型，支持8种图表
- **持续学习优化**：自动收集错误并修复，越用越聪明
- **Excel向量化**：支持Excel/CSV数据向量化，实现语义查询
- **实时状态管理**：提供完整的工作流状态跟踪和可视化
- **本地模型可用性改进**：即使API服务不可用，本地规则匹配功能仍然正常工作

### 1.3 目标用户
- 业务分析师：快速获取数据洞察
- 管理人员：自助式报表生成
- 数据运营：日常数据监控
- 产品经理：用户行为分析

## 2. 功能需求

### 2.1 智能查询功能

#### 2.1.1 查询类型支持

| 查询类型 | 功能描述 | 示例 | 优先级 |
|---------|---------|------|-------|
| **查找特定数据** | 查找最大值、最小值、排名等 | "哪个省公司的销售额最高" | P0 |
| **统计汇总** | 求和、计数、平均值、中位数、标准差等 | "统计各省份的平均值" | P0 |
| **筛选过滤** | 按条件筛选数据 | "筛选出金额大于1000的记录" | P0 |
| **排序** | 升序、降序排列 | "按销售额从高到低排序" | P1 |
| **复合查询** | 筛选+统计+排序组合 | "统计广东省的险情确认时长平均值" | P1 |
| **语义查询** | 基于向量化数据的语义理解查询 | "华东地区的销售额是多少" | P0 |

#### 2.1.2 高级统计函数

| 函数 | 描述 | 示例 |
|-----|------|------|
| sum | 求和 | "销售额的总和是多少" |
| avg | 平均值 | "销售额的平均值是多少" |
| max | 最大值 | "销售额的最大值是多少" |
| min | 最小值 | "销售额的最小值是多少" |
| count | 计数 | "有多少条记录" |
| median | 中位数 | "销售额的中位数是多少" |
| mode | 众数 | "哪个数值出现最多" |
| stdDev | 标准差 | "数据的波动程度如何" |
| variance | 方差 | "计算方差" |
| percentile | 百分位数 | "第90百分位数是多少" |
| quartile | 四分位数 | "第一四分位数是多少" |
| correlation | 相关系数 | "销售额和利润的相关性" |

#### 2.1.3 单位转换
- **时间单位**：秒→分钟、分钟→小时、小时→天
- **货币单位**：元→万元、千元→万元
- **自动识别**：系统根据用户输入自动识别转换需求

### 2.2 数据透视表功能

#### 2.2.1 核心能力
- **多维交叉分析**：支持行维度、列维度、值聚合
- **动态筛选**：支持按条件筛选数据
- **钻取功能**：点击单元格查看详细数据
- **多种聚合方式**：求和、平均、计数、最大、最小、中位数等

#### 2.2.2 支持的查询示例
- "按地区和产品交叉统计销售额"
- "做一个透视表分析各省各产品的销售情况"
- "按部门和月份交叉统计人数"

### 2.3 数据清洗功能

#### 2.3.1 支持的清洗操作

| 操作 | 描述 | 示例 |
|-----|------|------|
| 去重 | 删除重复行 | "删除重复数据" |
| 空值处理 | 填充、删除、插值 | "把空值填充为平均值" |
| 异常值检测 | IQR/Z-score方法 | "检测异常值" |
| 数据质量报告 | 完整性/唯一性分析 | "生成数据质量报告" |

### 2.4 可视化功能

#### 2.4.1 图表类型

| 图表类型 | 适用场景 | 交互特性 |
|---------|---------|---------|
| **柱状图** | 对比、排名、分类统计 | 悬停显示数值、点击筛选 |
| **折线图** | 趋势、时间序列 | 缩放、平移、数据点提示 |
| **饼图** | 占比、分布 | 扇区高亮、百分比显示 |
| **散点图** | 相关性分析 | 悬停显示坐标、趋势线 |
| **组合图** | 多指标对比 | 双Y轴支持 |
| **雷达图** | 多维度对比 | 多系列对比 |
| **漏斗图** | 转化分析 | 转化率显示 |
| **热力图** | 矩阵数据 | 颜色梯度显示 |

#### 2.4.2 智能图表推荐
系统基于数据特征自动推荐最佳图表类型：
- 时间序列数据 → 折线图
- 分类数据 → 柱状图
- 占比数据 → 饼图
- 相关性数据 → 散点图
- 多指标对比 → 组合图

### 2.5 意图识别系统

#### 2.5.1 三层混合架构

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：规则匹配引擎                                         │
│  - 响应时间：< 1ms                                           │
│  - 基于关键词和正则表达式                                     │
│  - 高置信度(>=0.8)时直接返回                                  │
├─────────────────────────────────────────────────────────────┤
│  第二层：本地BERT模型                                         │
│  - 响应时间：10-50ms                                          │
│  - 基于微调的BERT-base-chinese                               │
│  - 支持15种意图分类                                           │
├─────────────────────────────────────────────────────────────┤
│  第三层：大模型API兜底                                        │
│  - 响应时间：2-5秒                                            │
│  - 使用 SiliconFlow API (Kimi-K2.5)                         │
│  - 处理复杂、模糊的用户输入                                   │
└─────────────────────────────────────────────────────────────┘
```

#### 2.5.2 支持的意图类型（15种）

| 意图类型 | 标识符 | 描述 |
|---------|-------|------|
| 查找特定数据 | QUERY_FIND | 查找最大值、最小值、排名等 |
| 统计汇总 | QUERY_AGGREGATE | 求和、计数、平均值、中位数、标准差等 |
| 筛选过滤 | QUERY_FILTER | 按条件筛选数据 |
| 排序 | QUERY_SORT | 升序、降序排列 |
| 柱状图 | CHART_BAR | 柱状图可视化 |
| 折线图 | CHART_LINE | 折线图可视化 |
| 饼图 | CHART_PIE | 饼图可视化 |
| 通用图表 | CHART_GENERAL | 自动推荐图表类型 |
| 数据透视表 | PIVOT_TABLE | 多维交叉分析 |
| 数据清洗 | DATA_CLEAN | 去重、空值处理、异常检测 |
| 组合图表 | CHART_COMBO | 柱状图+折线图组合 |
| 雷达图 | CHART_RADAR | 多维度对比分析 |
| 漏斗图 | CHART_FUNNEL | 转化分析 |
| 热力图 | CHART_HEATMAP | 矩阵数据可视化 |
| 数据导出 | DATA_EXPORT | 导出Excel、CSV |

### 2.6 数据处理能力

#### 2.6.1 双模式数据处理

| 模式 | 数据量 | 技术方案 | 响应时间 |
|-----|-------|---------|---------|
| 内存模式 | < 10,000行 | JavaScript数组操作 | < 100ms |
| 数据库模式 | >= 10,000行 | SQL.js (SQLite) | < 500ms |

#### 2.6.2 文件支持
- **CSV文件**：支持UTF-8、GBK等编码，自动识别
- **Excel文件**：支持.xlsx和.xls格式
- **最大文件大小**：100MB
- **编码自动识别**：支持UTF-8、GBK等

### 2.7 数据导出功能

| 格式 | 描述 |
|-----|------|
| Excel (.xlsx) | 导出为Excel格式，支持多Sheet |
| CSV | 导出为CSV格式，支持中文 |
| 剪贴板 | 复制为表格格式，可直接粘贴到Excel |
| 报告 | 生成HTML/Markdown/文本格式报告 |

### 2.8 Excel向量化功能（V4.0新增）

#### 2.8.1 核心能力
- **数据向量化**：将Excel/CSV数据转换为向量存储
- **语义查询**：基于向量相似度的语义理解查询
- **实时状态更新**：显示"正在向量化"、"已向量化"、"未向量化"状态
- **语义扩展**：支持地域、时间、业务语义的理解

#### 2.8.2 支持的查询示例
- "华东地区的销售额是多少"
- "销售额最高的省份"
- "哪个产品的销量最好"

#### 2.8.3 技术实现
- **嵌入模型**：使用SentenceTransformer模型
- **向量存储**：使用ChromaDB
- **API接口**：提供向量化、查询、获取集合、删除集合等接口

### 2.9 实时状态管理（V5.0新增）

#### 2.9.1 核心能力
- **Agent工作流可视化**：实时显示查询处理的各个阶段状态
- **详细日志系统**：记录系统运行的详细信息，支持日志复制和清空
- **性能监控**：提供系统性能指标和并发状态监控
- **系统状态显示**：显示本地模型和大模型API的状态

#### 2.9.2 工作流步骤
1. **意图识别**：分析用户输入，确定查询意图
2. **实体提取**：提取查询中的关键实体和列名
3. **生成查询配置**：根据意图和实体生成查询配置
4. **执行查询**：执行数据查询和处理
5. **渲染结果**：生成图表或表格结果

### 2.10 本地模型可用性改进（V5.1新增）

#### 2.10.1 核心能力
- **本地规则匹配优先级**：即使API服务不可用，本地规则匹配功能仍然正常工作
- **状态显示优化**：系统设置中本地模型状态始终显示为可用
- **服务状态监控**：详细记录API服务状态，便于排查问题

#### 2.10.2 技术实现
- **状态判断逻辑优化**：修改`checkLocalModelAvailability`方法，即使API服务不可用，也将本地模型状态设为可用
- **服务状态记录**：保留对API服务可用性的检查，以便系统记录API服务状态
- **回调机制**：检测完成后触发回调，让UI更新状态显示

### 2.11 系统主动追问能力（V5.2新增）

#### 2.11.1 核心能力
- **智能意图理解**：当用户输入的查询意图不明确或信息不完整时，系统自动识别并发起追问
- **多维度追问**：支持对图表类型、数据维度、筛选条件等多个维度的追问
- **上下文理解**：基于用户历史输入和当前上下文，生成针对性的追问内容
- **用户友好提示**：使用自然语言生成清晰、易懂的追问提示

#### 2.11.2 支持的追问场景
- **图表类型不明确**：当用户请求图表但未指定具体类型时
- **数据维度缺失**：当用户请求分析但未指定具体维度时
- **筛选条件不足**：当用户请求筛选但条件不完整时
- **语义歧义**：当用户输入存在多种可能理解时

#### 2.11.3 技术实现
- **意图识别增强**：在三层混合架构的基础上，增加意图歧义检测能力
- **追问生成**：基于意图分析结果，生成针对性的追问内容
- **上下文管理**：维护用户会话上下文，确保追问的连贯性
- **状态管理**：在工作流中添加追问状态，支持追问-回答的完整流程

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 首屏加载时间 | < 3秒 | 包含所有JS资源加载 |
| 意图识别响应 | < 50ms | 本地BERT模型 |
| 查询执行时间 | < 500ms | 数据库模式 |
| 图表渲染时间 | < 1秒 | 包含数据处理和渲染 |
| 向量化响应 | < 5秒 | 基于数据量大小 |

### 3.2 可用性需求
- **浏览器兼容**：Chrome 90+、Firefox 88+、Edge 90+ 
- **响应式设计**：支持桌面端和移动端
- **离线可用**：核心功能支持离线使用（已加载数据后）
- **本地模型可靠性**：即使API服务不可用，本地规则匹配功能仍然可用

### 3.3 安全性需求
- **CORS配置**：支持跨域请求
- **输入验证**：防止恶意输入
- **错误处理**：优雅处理异常情况
