## 1. 产品概述

### 1.1 产品定位
智能数据分析助手是一款面向非技术用户的自然语言数据分析工具。用户无需掌握SQL或编程知识，只需用自然语言描述分析需求，系统即可自动理解意图、执行查询、生成可视化图表，并提供智能分析建议。

### 1.2 核心价值主张
- **零门槛数据分析**：自然语言交互，无需学习成本
- **智能意图理解**：三层混合架构，准确识别复杂需求
- **大数据支持**：自动切换数据库模式，支持百万级数据
- **一键可视化**：智能推荐最佳图表类型
- **持续学习优化**：自动收集错误并修复，越用越聪明

### 1.3 目标用户
- 业务分析师：快速获取数据洞察
- 管理人员：自助式报表生成
- 数据运营：日常数据监控
- 产品经理：用户行为分析

## 2. 功能需求

### 2.1 智能查询功能

#### 2.1.1 查询类型支持

| 查询类型 | 功能描述 | 示例 | 优先级 |
|---------|---------|------|-------|
| **查找特定数据** | 查找最大值、最小值、排名等 | "哪个省公司的销售额最高" | P0 |
| **统计汇总** | 求和、计数、平均值等 | "统计各省份的平均值" | P0 |
| **筛选过滤** | 按条件筛选数据 | "筛选出金额大于1000的记录" | P0 |
| **排序** | 升序、降序排列 | "按销售额从高到低排序" | P1 |
| **复合查询** | 筛选+统计+排序组合 | "统计广东省的险情确认时长平均值" | P1 |

#### 2.1.2 单位转换
- **时间单位**：秒→分钟、分钟→小时、小时→天
- **货币单位**：元→万元、千元→万元
- **自动识别**：系统根据用户输入自动识别转换需求

### 2.2 可视化功能

#### 2.2.1 图表类型

| 图表类型 | 适用场景 | 交互特性 |
|---------|---------|---------|
| **柱状图** | 对比、排名、分类统计 | 悬停显示数值、点击筛选 |
| **折线图** | 趋势、时间序列 | 缩放、平移、数据点提示 |
| **饼图** | 占比、分布 | 扇区高亮、百分比显示 |
| **散点图** | 相关性分析 | 悬停显示坐标、趋势线 |

#### 2.2.2 智能图表推荐
系统基于数据特征自动推荐最佳图表类型：
- 时间序列数据 → 折线图
- 分类数据 → 柱状图
- 占比数据 → 饼图
- 相关性数据 → 散点图

#### 2.2.3 图表配置
- X/Y轴自定义
- 聚合函数选择（sum、avg、count）
- 排序（升序/降序）
- 数据转换后展示

### 2.3 意图识别系统

#### 2.3.1 三层混合架构

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：规则匹配引擎                                         │
│  - 响应时间：< 1ms                                           │
│  - 基于关键词和正则表达式                                     │
│  - 高置信度(>=0.8)时直接返回                                  │
├─────────────────────────────────────────────────────────────┤
│  第二层：本地BERT模型                                         │
│  - 响应时间：10-50ms                                          │
│  - 基于微调的BERT-base-chinese                               │
│  - 支持8种意图分类                                            │
├─────────────────────────────────────────────────────────────┤
│  第三层：大模型API兜底                                        │
│  - 响应时间：2-5秒                                            │
│  - 使用 SiliconFlow API (Kimi-K2.5)                         │
│  - 处理复杂、模糊的用户输入                                   │
└─────────────────────────────────────────────────────────────┘
```

#### 2.3.2 支持的意图类型

| 意图类型 | 标识符 | 描述 |
|---------|-------|------|
| 查找特定数据 | QUERY_FIND | 查找最大值、最小值、排名等 |
| 统计汇总 | QUERY_AGGREGATE | 求和、计数、平均值等 |
| 筛选过滤 | QUERY_FILTER | 按条件筛选数据 |
| 排序 | QUERY_SORT | 升序、降序排列 |
| 柱状图 | CHART_BAR | 柱状图可视化 |
| 折线图 | CHART_LINE | 折线图可视化 |
| 饼图 | CHART_PIE | 饼图可视化 |
| 通用图表 | CHART_GENERAL | 自动推荐图表类型 |

#### 2.3.3 拒识机制
系统能够识别并拒绝非数据分析相关的输入：
- 问候语（你好、谢谢等）
- 无关查询（天气、新闻、股票等）
- 过短输入（少于3个字符）
- 纯数字或符号输入

#### 2.3.4 需求分类器
- **精准模式**：列名匹配度高、需求明确，使用本地模型处理
- **智能模式**：模糊度高、复杂需求，调用大模型处理
- **自动路由**：根据输入特征自动选择处理模式

### 2.4 数据处理能力

#### 2.4.1 双模式数据处理

| 模式 | 数据量 | 技术方案 | 响应时间 |
|-----|-------|---------|---------|
| 内存模式 | < 10,000行 | JavaScript数组操作 | < 100ms |
| 数据库模式 | >= 10,000行 | SQL.js (SQLite) | < 500ms |

#### 2.4.2 SQL.js数据库功能
- 数据表创建与索引
- SQL查询执行（筛选、聚合、排序、分页）
- 极值查找（MAX/MIN）
- 统计信息计算（count、avg、sum等）
- 重复列名自动处理

#### 2.4.3 数据预处理器
- **单位转换**：秒→分钟、元→万元等
- **公式计算**：同比增长率、环比增长率、占比、完成率、利润率
- **数据格式化**：千分位分隔符、小数位数控制、百分比格式
- **空值处理**：自定义空值显示

#### 2.4.4 文件支持
- **CSV文件**：通过PapaParse解析
- **Excel文件**：通过SheetJS/xlsx解析
- **最大文件大小**：100MB
- **编码自动识别**：支持UTF-8、GBK等

### 2.6 错误处理与自动修复

#### 2.6.1 错误收集器
- **全局错误捕获**：自动捕获未处理的JavaScript错误和Promise rejection
- **错误分类**：COLUMN_ERROR、PARAM_ERROR、DATA_ERROR、NETWORK_ERROR等
- **localStorage持久化**：最大存储1000条错误记录
- **错误报告导出**：支持下载JSON格式的错误报告

#### 2.6.2 自动修复器

| 规则ID | 规则名称 | 功能描述 |
|-------|---------|---------|
| semantic_mapping_missing | 语义映射缺失自动补充 | 自动添加新术语到语义映射表 |
| low_confidence_intent | 低置信度意图识别优化 | 记录低置信度案例用于模型优化 |
| param_extraction_failed | 参数提取失败修复 | 使用大模型API重新提取参数 |
| chart_type_mismatch | 图表类型不匹配修复 | 重新推荐合适的图表类型 |
| data_format_error | 数据格式错误修复 | 自动修正数据格式 |
| user_feedback_driven | 用户反馈驱动的修复 | 根据用户反馈调整识别策略 |

- **定时修复**：每60分钟自动执行一次
- **手动触发**：支持一键修复所有错误

#### 2.6.3 错误引导UI
- 错误发生时显示友好的引导提示
- 提供可能的解决方案
- 一键反馈错误

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 首屏加载时间 | < 3秒 | 包含所有JS资源加载 |
| 意图识别响应 | < 50ms | 本地BERT模型 |
| 查询执行时间 | < 500ms | 数据库模式 |
| 图表渲染时间 | < 1秒 | 包含数据处理和渲染 |
| 并发用户数 | 10 | 本地部署场景 |

### 3.2 可用性需求
- **浏览器兼容**：Chrome 90+、Firefox 88+、Edge 90+
- **响应式设计**：支持桌面端和移动端
- **离线可用**：核心功能支持离线使用（已加载数据后）
- **错误恢复**：系统崩溃后可恢复未完成的分析

### 3.3 安全需求
- **数据安全**：用户数据不上传服务器，本地处理
- **API密钥保护**：配置文件不提交到版本控制
- **XSS防护**：所有用户输入进行转义处理
- **CSRF防护**：API请求验证来源

### 3.4 可扩展性需求
- **模块化架构**：新功能以模块形式添加
- **插件机制**：支持第三方Skills扩展
- **配置驱动**：意图识别规则可配置
- **模型热更新**：BERT模型可在线更新
