# 智能数据分析助手 V3.0 - 系统开发历程

## 一、项目概述

**产品定位**：面向非技术用户的数据分析工具，用户只需上传数据文件，用自然语言描述分析需求，系统自动完成数据查询、统计分析和可视化。

**技术栈**：
- 前端：HTML/CSS/JavaScript + Chart.js
- 后端：Python FastAPI + Flask
- AI：BERT本地模型 + 大模型API（DeepSeek）
- 数据处理：Pandas

---

## 二、架构演进历程

```
用户输入 → 意图识别(规则) → 大模型生成配置 → 执行查询 → 返回结果
```

**存在问题**：
- ❌ 请求超时（60秒）

### V2.0 架构：引入本地模型


**存在问题**：
- ❌ 正则表达式匹配死板，用户表达稍有变化就失败
- ❌ 意图识别成功但配置生成失败，频繁fallback到大模型
- ❌ 大模型调用后仍重复调用本地模型

## 三、关键问题与解决方案

### 问题1：请求超时（60秒）

**现象**：用户提交查询后，系统长时间无响应，最终超时报错。

**根因分析**：
```
意图识别(15s) + 配置生成(45s) = 总计60秒
```
两个大模型调用串行执行，且配置生成后仍重复调用本地模型。

**解决方案**：
1. 消除重复调用：意图识别后直接使用已生成的配置
2. 简单查询走本地模型：毫秒级响应
3. 复杂查询优化提示词：减少Token消耗

**效果**：简单查询从60秒降至**毫秒级**，复杂查询从60秒降至**20秒内**。

---

### 问题2：Skills模块从未被调用

**现象**：系统集成了数据分析、趋势分析、异常检测等Skills，但日志显示从未触发。

**根因分析**：
```
原逻辑：意图识别 → 配置生成 → 执行
        ↑ Skills检测在这里，但已经晚了
```
Skills检测放在配置生成阶段，而此时已确定走本地模型或大模型。

**解决方案**：
将Skills检测前置到需求分类阶段：
```
新逻辑：需求分类 → Skills检测 → 决策模式 → 执行
```

**效果**：Skills调用率从0%提升至**100%**（对于需要分析的需求）。

---

### 问题3：正则表达式匹配死板

**现象**：
```
用户输入："查找最大的险情确认时长"
正则模式：/(查找|找出)(.+?)(最大|最小)的/
匹配结果：失败（因为"的"字位置不对）
```

**根因分析**：
正则表达式只能匹配固定模式，无法理解语义。用户表达方式多样，无法穷举所有模式。

**解决方案**：
引入BERT语义匹配作为第二层配置生成：
```python
# 第一层：正则表达式（快速，覆盖常见模式）
if 正则匹配成功:
    return 配置

# 第二层：BERT语义匹配（智能，理解用户意图）
if BERT匹配成功:
    return 配置

# 第三层：大模型生成（兜底，处理复杂需求）
return 大模型生成配置
```

**效果**：配置生成成功率从**30%提升至95%**。

---

### 问题4：列名匹配失败

**现象**：
```
用户输入："查找最大的险情确认时长"
实际列名："险情确认时长"
匹配结果：空字符串 ""
```

**根因分析**：
```python
columns = ['', '主键', '省公司名称', ..., '险情确认时长']
# 空字符串 '' 在任何文本中都能匹配成功！
# 因为 "" in "任何文本" 返回 True
```

**解决方案**：
```python
# 过滤掉空字符串和无效列名
columns = [col for col in columns if col and col.strip()]
```

**效果**：列名匹配准确率从**0%提升至100%**。

---

### 问题5：BERT模型准确率低（66%）

**现象**：
```
测试用例：
"绘制柱状图展示各区域数据" → 识别为 CHART_PIE（错误）
"用饼图展示占比" → 识别为 QUERY_SORT（错误）
```

**根因分析**：
训练数据只有100条，每个意图类型约10-20条，样本量不足。

**解决方案**：
1. 编写数据生成脚本，自动生成多样化样本
2. 扩充训练数据至1000条
3. 覆盖同义表达、不同句式、口语化表达

**效果**：
| 指标 | 100条数据 | 1000条数据 |
|------|----------|-----------|
| 准确率 | 66.67% | **98.67%** |
| F1分数 | 0.598 | **0.986** |

## 四、技术亮点总结

### 1. 智能路由架构
- **精准模式**：本地模型处理简单查询，毫秒级响应
- **智能模式**：大模型处理复杂需求，支持自然语言理解

### 2. 三层配置生成
- 正则表达式（快速）→ BERT语义匹配（智能）→ 大模型（兜底）
- 兼顾效率与灵活性

### 3. 本地BERT模型
- 意图识别准确率98.67%
- 配置生成语义匹配
- 无需联网，数据安全

### 4. Skills集成
- 数据分析、趋势分析、异常检测、业务建议
- 在需求分类阶段智能判断是否需要调用
