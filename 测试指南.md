## 一、测试环境准备

### 1.1 启动本地服务器

```bash
# 在项目根目录下执行
python -m http.server 8000
```

### 1.2 访问测试页面

打开浏览器访问：
- **主应用**: http://localhost:8000/index.html
- **测试中心**: http://localhost:8000/test.html

## 二、测试中心使用说明

### 2.1 访问测试中心

1. 确保本地服务器已启动
2. 打开浏览器访问 `http://localhost:8000/test.html`
3. 等待所有模块加载完成

### 2.2 测试概览

测试中心提供以下测试类型：

- **单元测试** (5 个模块)
  - 图表推荐器
  - 错误收集器
  - 反馈管理器
  - 数据预处理器
  - 自动修复器

- **集成测试** (2 个)
  - 完整查询流程
  - 模块间调用

- **UI 测试** (3 个)
  - 反馈组件
  - 错误引导弹窗
  - 错误报告查看器

- **性能测试** (2 个)
  - 模块加载时间
  - 响应时间

### 2.3 运行测试

#### 方式一：单个测试

点击各个测试卡片上的"运行测试"按钮

#### 方式二：批量测试

依次点击所有测试按钮（建议按顺序）

#### 方式三：控制台测试

打开浏览器控制台（F12），执行：

```javascript
// 运行所有单元测试
runChartRecommenderTest();
runErrorCollectorTest();
runFeedbackManagerTest();
runDataPreprocessorTest();
runAutoFixerTest();

// 运行 UI 测试
runFeedbackUITest();
runErrorGuideUITest();
runErrorReportUITest();

// 运行性能测试
runLoadTimeTest();
runResponseTimeTest();
```

## 三、测试用例详解

### 3.1 图表推荐器测试

**测试目标**：验证图表智能推荐功能

**测试输入**：
```javascript
dataInfo: {
    columns: ['省公司', '日期', '金额', '数量'],
    sampleData: [],
    rowCount: 100
}
intent: 'QUERY_AGGREGATE'
userInput: '统计各省公司的平均金额'
```

**预期结果**：
- ✅ 返回至少 1 种推荐图表
- ✅ 推荐类型为柱状图或饼图
- ✅ 置信度 > 0.5

**通过标准**：`recommendations.length > 0`

---

**测试输入**：
```javascript

    userInput: '测试',
    module: 'test',
    action: 'run_test'

```

**预期结果**：

- ✅ ID 格式：`ERR_timestamp_random`
- ✅ 错误保存到 localStorage

**通过标准**：`errorId.startsWith('ERR_')`

**预期结果**：

### 3.4 数据预处理器测试

**测试目标**：验证数据转换功能

**测试输入**：
```javascript
testData: [
    { 省公司：'广东', 金额：1500000 },
    { 省公司：'北京', 金额：1300000 }
]

    unitConversion: {
        from: '元',
        to: '万元',
        formula: (v) => v / 10000,
        decimalPlaces: 2
    }
}
```

**预期结果**：
- ✅ 1500000 元 → 150 万元
- ✅ 1300000 元 → 130 万元

**通过标准**：`result[0].金额 === 150`

## 五、常见问题排查

### Q1: 测试页面无法打开？

**A**: 检查以下几点：
1. 确认本地服务器已启动
2. 检查端口 8000 是否被占用
3. 尝试访问 http://127.0.0.1:8000/test.html

### Q2: 模块加载失败？

**A**: 打开浏览器控制台（F12），查看错误信息：
- 如果是 404 错误，检查文件路径
- 如果是语法错误，检查浏览器兼容性
- 确保使用现代浏览器（Chrome 80+、Firefox 75+）

### Q3: 测试结果显示失败？

**A**: 
1. 查看测试日志中的详细错误信息
2. 检查 localStorage 是否可用
3. 清除浏览器缓存后重试

### Q4: UI 测试没有反应？

**A**:
1. 确认 CSS 文件已正确加载
2. 检查控制台是否有 JavaScript 错误
3. 确认 feedbackUI.init() 已调用
